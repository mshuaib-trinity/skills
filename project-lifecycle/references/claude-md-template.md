# CLAUDE.md Template

This is the source for the project's `CLAUDE.md` and its identical mirror `AGENTS.md`.
The generated file is **self-contained**: an AI agent reads it and knows every
lifecycle rule without opening the skill. Inline the rules — do not just point elsewhere.

## How to apply this template

**If `CLAUDE.md` / `AGENTS.md` do NOT exist:**
Fill in every `{{PLACEHOLDER}}` and write the full template below to both files.

**If `CLAUDE.md` / `AGENTS.md` already exist:**
Do not overwrite. Prepend the **"Read First" through "Key Guides"** block (everything
between the two `<!-- LIFECYCLE: ... -->` markers) to the **top** of each existing file,
directly under the first H1. Leave the project's existing content untouched below it.
Then reconcile: if the existing file already had a Task Management section, merge — keep
the stricter rule, delete the weaker duplicate. The two files must end identical.

**Always, after writing:**
```bash
cp CLAUDE.md AGENTS.md          # or hand-mirror if CLAUDE.md has Claude-only notes
diff CLAUDE.md AGENTS.md        # must produce no output
```

## Task Management Protocol

**Automatic. No user trigger required. No exceptions.**

### Before Any Work

1. Read `tasks/ACTIVE.md`.
2. If the work is not tracked → create a task in `tasks/future/` first.
3. Move it to `tasks/current/` and update `tasks/ACTIVE.md` when work begins.
4. **Do not write code before the task exists in `tasks/current/`.**

A planning, brainstorming, or grilling session is itself work: create the task first,
then put its `prd.md`, `brainstorm.md`, and `plan.md` **inside that task/epic directory**.
PRDs are task- or epic-specific.

### Task Types

| Type | Lives in | Requires |
|------|----------|----------|
| Epic | `tasks/{future,current,completed}/epic-<slug>/` | `task.md`, `prd.md`, `kanban.md`, `tasks/` subdirectory |
| Standalone | `tasks/{future,current,completed}/task-<slug>/` | `task.md` only |

**Epics require a `prd.md`** — write it before starting any task within the epic.

### During Work

- Update `tasks/ACTIVE.md` whenever task state changes (started, blocked, completed).
- Update `tasks/STATUS.md` — set the **Last updated** line on every state change.
- Update `kanban.md` immediately when any epic task changes status — do not wait.
- Update the relevant `docs/` file the moment code changes — not after.
  An undocumented change is an incomplete change. Triggers: [`docs/NAVIGATION.md`](docs/NAVIGATION.md).
- Out-of-scope discoveries → `tasks/backlog/discovered-issues.md` or a new task. Never silently absorbed.

### Completion Checklist

Run before closing any task. All applicable items must be checked. No exceptions.

- [ ] 1. `task.md` — set `status: completed`
- [ ] 2. `kanban.md` — update row to ✅ completed (epics only)
- [ ] 3. `tasks/ACTIVE.md` — move task out of In Progress; update Up Next
- [ ] 4. `tasks/STATUS.md` — update Last updated line
- [ ] 5. `docs/` — update every file affected by the code change (see Doc Trigger Table)
- [ ] 6. `docs/adr/` — write an ADR if an architectural decision was made ([format](docs/adr/FORMAT.md))
- [ ] 7. `CLAUDE.md` + `AGENTS.md` — update identically if any rule or contract changed
- [ ] **Run:** `python scripts/validate-project.py` — must pass

**Folder moves (mandatory, immediate — no deferral):**
- Completed task → `tasks/completed/`
- Completed epic (all tasks done) → `tasks/completed/`
- New epic starting → move from `tasks/future/` to `tasks/current/`
